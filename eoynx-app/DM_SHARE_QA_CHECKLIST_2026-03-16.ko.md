# DM 공유 QA 체크리스트 (2026-03-16)

## 범위
- 웹 피드 카드 공유 모달 -> DM 전송
- 모바일 피드 공유 -> DM 전송
- DM 수신함/스레드에서 공유 메시지 렌더링

## 사전 조건
- 테스트 계정 A/B 준비
- A가 B를 팔로우
- A/B 모두 암호화 키 초기화 완료
- A/B 모두 로그인 상태

## 웹 시나리오
- [x] 로그인 사용자가 피드 카드에서 공유 버튼 클릭 시 공유 모달이 열린다.
- [x] DM 모드 진입 시 팔로잉 목록이 로드된다.
- [x] 사용자 선택 후 전송 버튼 활성화 및 선택 인원 수가 표시된다.
- [x] DM 전송 성공 시 성공 알림이 노출된다.
- [x] DM 전송 실패 시 에러 알림이 노출된다.
- [x] 전송 후 상대방 DM 수신함/스레드에 공유 메시지가 표시된다.
- [x] 비로그인 상태에서 공유 버튼 클릭 시 인증 페이지로 이동한다.

## 모바일 시나리오
- [x] 피드에서 DM 공유 대상 선택 후 전송이 성공한다.
- [x] DM 요청 필요 계정에 전송 시 정책 알림(요청 필요)이 표시된다.
- [x] 전송 후 수신함 프리뷰/스레드 본문이 정상 렌더링된다.

## 회귀 점검
- [ ] 피드 좋아요/북마크/댓글 기능 영향 없음
- [ ] 알림 카운트/알림 목록 갱신 영향 없음
- [ ] DM 이미지 전송 기능 영향 없음

## 메모
- 웹 서버가 production 모드(next start)로 실행 중일 때는 변경 반영을 위해 build + restart가 필요함.
- 현재 공유 메시지 전송은 텍스트 암호화(encryptedContent + iv) 경로를 사용함.
